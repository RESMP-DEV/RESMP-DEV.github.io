Automating Evidence-Based Medical Content

This project demonstrates a state-of-the-art Retrieval-Augmented Generation (RAG) pipeline designed to automate the creation of scientifically-grounded medical articles. By programmatically sourcing and citing information from the trusted NCBI PubMed database, the system can generate comprehensive articles on complex medical topics, complete with relevant, AI-generated medical illustrations. This powerful prototype proves the core technical feasibility of leveraging generative AI to rapidly produce accurate, evidence-based medical content, significantly reducing manual research and writing time.

Medical Article Generator Summary

Stage 1: Ingest & Index

This is the foundational, offline phase where the system builds its knowledge. It involves programmatically fetching thousands of articles from PubMed, breaking them into smaller, digestible chunks, and then using an embedding model to convert each chunk into a numerical vector. These vectors, which capture the semantic meaning of the text, are then stored and indexed in the Weaviate vector database, creating a searchable library of medical knowledge.

Stage 2: Retrieve & Augment

When a user submits a query (e.g., "articles on diabetic retinopathy"), this stage begins. The system converts the user's query into a vector and uses it to search the Weaviate database. It retrieves the most semantically similar text chunks from the indexed articles. This retrieved information forms the "context" that will be used to ground the AI's response in factual, verifiable data. This is the "Retrieval" in RAG.

Stage 3: Generate & Synthesize

In the final stage, the retrieved context and the original user query are passed to a Large Language Model (LLM). The LLM's task is not to answer from its general knowledge, but to synthesize the provided context into a comprehensive, well-written article. Simultaneously, a prompt is generated for the image model to create a relevant illustration. The final output is a complete article, grounded in the source data and enhanced with a custom visual.
